# 系统架构说明

## 1. 总体架构

采用分层单体架构（便于毕业设计答辩展示与部署）：

- **表示层**：Thymeleaf 页面 + REST API
- **业务层**：TraceabilityService / BlockchainEvidenceService
- **数据层**：MyBatis Mapper + 关系数据库（H2/MySQL）
- **可信扩展层**：链式哈希存证（可替换为联盟链）

## 2. 角色与职责

- **生产者**：创建批次、记录生产环节信息
- **加工/物流企业**：补充加工工艺、运输轨迹与冷链参数
- **监管部门**：查看全链条、抽检记录与异常追踪
- **消费者**：扫码查看来源、流转、检测等信息

## 3. 关键设计点

### 3.1 全生命周期数据模型
- product_batch：批次基础信息
- trace_event：链路事件（阶段、地点、人员、详情、IoT 数据）

### 3.2 防篡改机制
每条 trace_event 保存：
- `prev_hash`：上一条记录哈希
- `block_hash`：当前记录哈希（SHA-256）

形成单批次内部链路，支持完整性校验。

### 3.3 区块链接口预留
BlockchainEvidenceService 目前返回模拟交易号，后续可改造：
- 调用 Fabric SDK
- 将关键字段写入链上
- 保存链上 txId 到 `on_chain_tx_id`

## 4. 非功能目标

- **可用性**：低成本、易部署
- **可扩展性**：后续可拆微服务
- **安全性**：关键数据哈希校验
- **可维护性**：清晰分层、接口化设计
